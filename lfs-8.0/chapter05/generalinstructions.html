5.3. 通用编译指南

在使用本指南构建软件包时，有几个约定：

  • 有的软件包需要在编译之前打补丁，以此来规避一些问题。有的补丁会在本章和下一章中都需要， 但有的只有其中一章会需要。因此，如果某章看起来缺少某个补丁的下载说明时不用担心。 打补丁的时候还可能会遇到关于 offset 或 fuzz 的警告信息。别担心这些警告，补丁还是会成功安装的。

  • 在大部分软件包的编译过程中，屏幕上都可能出现一些警告信息。这很正常， 并且可以被安全地忽略。那些警告正如它们描述的那样—是对过时但合法的 C 或 C++ 语法的警告。C 语言的标准经常改变，然而一些软件包仍然在使用旧的标准。 这并不是一个问题，虽然确实会抛出警告。

  • 最后一次确认是否正确设置了 LFS 环境变量： 

    echo $LFS

    确认输出显示的是 LFS 分区挂载点的路径，在我们的例子中也就是 /mnt/lfs。

  • 最后，有两个重要的点必须要强调一下：

    重要

    编译指南假定你已经正确地设置了宿主系统 （包括符号链接）：

    • shell 使用的是 bash。

    • sh 是到 bash 的符号链接。（译注：符号链接又称软链接）

    • /usr/bin/awk 是到 gawk 的符号链接。

    • /usr/bin/yacc 是到 bison 的符号链接或者一个执行 bison 的小脚本。

    重要

    再次强调构建过程：

    1. 把所有源文件和补丁放到 chroot 环境可访问的目录，例如 /mnt/lfs/sources/。 不要 把源文件放到 /mnt/lfs/tools/ 目录下。

    2. 进入到源文件所在的目录中。

    3. 对于每个软件包：

      1. 用 tar 命令解压要编译的软件包。在第五章中，确保解压软件包时你是 lfs 用户。

      2. 进入到解压后创建的目录中。

      3. 根据指南说明编译软件包。

      4. 回退到源文件目录。

      5. 除非特别说明，删除解压出来的源目录。

4.2. 创建 $LFS/tools 文件夹

第五章中编译的所有程序将安装在 $LFS/tools 文件夹下，以便和第六章中编译的程序分离开来。这里编译的程序是临时工具，并非最终 LFS 系统的一部分。将这些编译后的临时工具保存在单独的目录，以方便在使用完成后直接删除。同时也防止了和最终的 LFS 系统相互冲突。（第五章的工作很容易出现一些意料不到的问题。）

使用 root 用户，通过以下命令来创建所需目录： 

mkdir -v $LFS/tools

下一步就是在系统中创建一个 /tools 文件夹的符号链接，该链接指向 LFS 分区上创建的新的目录。使用 root 用户来执行此命令： 

ln -sv $LFS/tools /

注意

上述命令是正确的。 ln 命令有一些语法上的变化，会检查 info coreutils ln 和 ln(1) 然后报告一个错误。

创建的符号链接用来编译工具链， The created symlink enables the toolchain to be compiled so that it always refers to /tools, meaning that the compiler, assembler, and linker will work both in Chapter 5 (when we are still using some tools from the host) and in the next (when we are “chrooted” to the LFS partition).

4.3. 添加 LFS 用户

当使用 root 用户登陆时，一点小的错误就有可能破坏整个系统。因此，我们建议使用普通用户（非特权用户）构建这些软件包。你可以使用自己的用户名，但是为了更容易配置所需的工作环境，请创建一个名为 lfs 的新用户组，并创建名字同为 lfs 的新用户作为该用户组的成员，然后在安装过程中将会使用此用户。以 root 用户，键入以下命令： 

groupadd lfs
useradd -s /bin/bash -g lfs -m -k /dev/null lfs

命令中选项的含义：

-s /bin/bash

设置 bash 为 lfs 用户的默认 shell；

-g lfs

将 lfs 用户添加到 lfs 用户组；

-m

为 lfs 用户创建 home 文件夹；

-k /dev/null

这个选项是通过将输入位置更改为 /dev/null, 从而阻止一些可能从默认的 /etc/skel 目录复制过来文件；

lfs

要创建用户的实际名称。

为 lfs 用户设置密码： 

passwd lfs

将 $LFS/tools 目录的所有者更改为 lfs 用户，使 lfs 用户获取完全访问权限： 

chown -v lfs $LFS/tools

如果按照建议创建了单独的工作目录，请使用以下命令将该目录的拥有者更改为 lfs 用户： 

chown -v lfs $LFS/sources

接下来以 lfs 用户登陆。可以通过虚拟控制台 (Virtual Console) 、显示管理器 (Display Manager) 或以下命令切换至 lfs 用户： 

su - lfs

“-” 是指完整切换到 lfs 用户环境，即 login shell ；如果不加 “-” 则表示只切换到其他用户，即 non-login shell 。这两种切换用户的区别可以在 bash(1) 和 info bash 中找到。

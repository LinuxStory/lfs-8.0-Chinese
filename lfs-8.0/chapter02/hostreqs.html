2.2. 宿主系统需求

你的宿主系统应该拥有这些软件，并且其版本号不应低于以下列出的。 对于大多数现代的 Linux 发行版来说，这不应该成为一个问题。 另外还需要注意，许多发行版会经常以 “<package-name>-devel” 或者 “<package-name>-dev” 的形式， 将软件的头文件放在分开的软件包中。如果你使用的发行版提供这类软件包的话， 请记得安装它们。

以下列出的软件包的更早版本可能仍然能够符合预期地工作，但是这并没有被测试过。

  • Bash-3.2 （/bin/sh 应该是一个指向 bash 的符号链接或者硬链接）

  • Binutils-2.17 （不推荐 2.27 以后的版本，因为它们并没有被测试过）

  • Bison-2.3 （/usr/bin/yacc 应该是一个指向 bison 的链接，或者是一个运行 bison 的小脚本）

  • Bzip2-1.0.4

  • Coreutils-6.9

  • Diffutils-2.8.1

  • Findutils-4.2.31

  • Gawk-4.0.1 (/usr/bin/awk 应该是一个指向 gawk 的链接)

  • GCC-4.7 包括 C++ 编译器，g++（不推荐 6.3.0 以后的版本， 因为它们并没有被测试过）

    说明

    有报告称，在一些发行版上，GCC 所使用的库可能出现不一致的状态， 并且已经影响到了一些 LFS 包的构建。可以检查一下 /usr/lib 目录下 （可能的话再检查 /usr/lib64）是否有 libgmp.la, libmpfr.la，和 libmpc.la。 这三个文件或者都存在，或者都不存在，不应该是只有一个或者两个的情况。 如果在你的系统上出现了这个问题，可以重命名或者删除这些 .la 文件， 或者安装它们所对应的缺失了软件包。

  • Glibc-2.11 （不推荐 2.25 以后的版本，因为它们并没有被测试过）

  • Grep-2.5.1a

  • Gzip-1.3.12

  • Linux Kernel-2.6.32

    之所以要求这个内核版本，是因为在第 6 章中构建 glibc 时，我们指定这个版本作为开发者的推荐版本。 同样，udev 也依赖于这个版本。

    如果你的宿主内核比 2.6.32 早的话，你需要将你的内核替换为更新的内核。 有两种方式可以达到目的。首先检查一下你的 Linux 维护者是否已经提供了 2.6.32 或者更新的内核包。如果是的话，你不妨安装它。 如果你的维护者没有提供一个可以接受的内核包，或者你不太想安装它的话， 你可以自己编译它。在 第 8 章 中有关于编译内核和配置引导器（假定宿主机使用 GRUB）的指导。

  • M4-1.4.10

  • Make-3.81

  • Patch-2.5.4

  • Perl-5.8.8

  • Sed-4.1.5

  • Tar-1.22

  • Texinfo-4.7

  • Xz-5.0.0

注意

注意，本书中构建 LFS 使用的指令都假定你使用了上面提到的符号链接。 指向其它软件的符号链接（例如 dash, mawk 等等）可能符合预期地工作， 但是这并没有被测试过，也可能不被 LFS 开发团队所支持， 并且可能需要使用不同于书中的描述的指令，或者对一些软件包的额外的补丁。

运行以下命令，以检查你的宿主系统是否拥有所有合适的版本，以及编译程序的能力。 

cat > version-check.sh << "EOF"
#!/bin/bash
# Simple script to list version numbers of critical development tools
export LC_ALL=C
bash --version | head -n1 | cut -d" " -f2-4
MYSH=$(readlink -f /bin/sh)
echo "/bin/sh -> $MYSH"
echo $MYSH | grep -q bash || echo "ERROR: /bin/sh does not point to bash"
unset MYSH

echo -n "Binutils: "; ld --version | head -n1 | cut -d" " -f3-
bison --version | head -n1

if [ -h /usr/bin/yacc ]; then
  echo "/usr/bin/yacc -> `readlink -f /usr/bin/yacc`";
elif [ -x /usr/bin/yacc ]; then
  echo yacc is `/usr/bin/yacc --version | head -n1`
else
  echo "yacc not found" 
fi

bzip2 --version 2>&1 < /dev/null | head -n1 | cut -d" " -f1,6-
echo -n "Coreutils: "; chown --version | head -n1 | cut -d")" -f2
diff --version | head -n1
find --version | head -n1
gawk --version | head -n1

if [ -h /usr/bin/awk ]; then
  echo "/usr/bin/awk -> `readlink -f /usr/bin/awk`";
elif [ -x /usr/bin/awk ]; then
  echo awk is `/usr/bin/awk --version | head -n1`
else 
  echo "awk not found" 
fi

gcc --version | head -n1
g++ --version | head -n1
ldd --version | head -n1 | cut -d" " -f2-  # glibc version
grep --version | head -n1
gzip --version | head -n1
cat /proc/version
m4 --version | head -n1
make --version | head -n1
patch --version | head -n1
echo Perl `perl -V:version`
sed --version | head -n1
tar --version | head -n1
makeinfo --version | head -n1
xz --version | head -n1

echo 'int main(){}' > dummy.c && g++ -o dummy dummy.c
if [ -x dummy ]
  then echo "g++ compilation OK";
  else echo "g++ compilation failed"; fi
rm -f dummy.c dummy
EOF

bash version-check.sh

同时检查一些库的一致性： 

cat > library-check.sh << "EOF"
#!/bin/bash
for lib in lib{gmp,mpfr,mpc}.la; do
  echo $lib: $(if find /usr/lib* -name $lib|
               grep -q $lib;then :;else echo not;fi) found
done
unset lib
EOF

bash library-check.sh

在这个脚本中列出的这三个文件或者都存在，或者都不存在， 不应该是只存在一个或者两个的情况。

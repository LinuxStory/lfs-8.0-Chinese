6.43. Autoconf-2.69

Autoconf 软件包包含了用于生成自动配置源代码 shell 脚本的程序。

预计构建时间: 少于 0.1 SBU (加上测试大约 3.5 SBU)
需要磁盘空间: 17.3 MB

6.43.1. Autoconf 安装过程

准备编译 Autoconf: 

./configure --prefix=/usr

编译软件包: 

make

测试编译结果，输入: 

make check

这一步将需要很长一段时间，大约 3.5 SBUs。另外，使用 Automake 的部分 测试被跳过了。要全面测试覆盖率，可以在 Automake 安装之后重新测试 Autoconf。 另外，libtool-2.4.3 和之后版本的变化会导致两个测试失败。

安装软件包: 

make install

6.43.2. Autoconf 的内容

安装的程序: autoconf, autoheader, autom4te, autoreconf, autoscan, autoupdate, and ifnames
安装的目录: /usr/share/autoconf

简要介绍

autoconf

生成用于配置软件源代码以适应不同的类 Unix 系统的 shell 脚本; 生成的配置脚本是独立运行的——运行这些脚本不需要 autoconf 程序

autoheader

用于生成 C 语言 #define 语句模板文件给 configure 用的工具

autom4te

一个 M4 宏处理的封装

autoreconf

当修改了 autoconf 和 automake 模板文件时， 以正确的顺序自动运行 autoconf, autoheader, aclocal, automake, gettextize, 和 libtoolize 来节约时间。

autoscan

为软件包协助创建一个configure.in 文件; 它检查目录树中的源文件，检查他们中是否有一般的可移植性问题， 然后创建一个 configure.scan 文件 用于作为软件包最初的 configure.in文件

autoupdate

修改仍在使用旧的 autoconf 宏的 configure.in 文件去使用现在的宏名

ifnames

协助编写软件包的 configure.in 文件; [它打印出软件包在 C 预处理器条件中使用的标识符[如果一个软件包 已经为一定的可移植性设置了，这个程序将帮助决定 configure 需要检查什么。它也可以在 autoscan 生成的 configure.in 文件中填空。]

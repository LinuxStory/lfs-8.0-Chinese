6.53. Findutils-4.6.0

Findutils 软件包包含用来查找文件的工具。这些工具可以在目录树中进行递归查找，并创建、维护和搜索数据库（一般会比递归查找快，但是如果不经常更新数据库的话结果不可靠）。

大概编译时间： 0.9 SBU
所需磁盘空间： 48 MB

6.53.1. 安装 Findutils

首先，关闭在某些机器上会无限循环的测试： 

sed -i 's/test-lock..EXEEXT.//' tests/Makefile.in

为编译 Findutils 做准备： 

./configure --prefix=/usr --localstatedir=/var/lib/locate

配置选项的含义：

--localstatedir

这个选项把locate 数据库的位置改到 /var/lib/locate目录内，使其兼容 FHS 标准。

编译软件包： 

make

测试编译结果，执行： 

make check

安装软件包： 

make install

LFS-Bootscripts 软件包中的一些脚本依赖于 find. 由于/usr 在启动前期有可能不可用，这个工具需要放在根分区上。为了修正一个路径 updatedb 脚本也需要修改： 

mv -v /usr/bin/find /bin
sed -i 's|find:=${BINDIR}|find:=/bin|' /usr/bin/updatedb

6.53.2. Findutils 的内容

安装的程序 code, find, locate, oldfind, updatedb, and xargs

简介

code

之前被用来处理locate 数据库；它是 frcode 的前身

find

在给定的目录树中搜索匹配指定条件的文件

locate

在一个包含文件名的数据库中搜索并报告包含给定字符串或匹配给定模式的文件名

oldfind

老版本的 find，使用了不同的算法

updatedb

更新 locate 数据库；它会扫描整个文件系统（包括当前被挂载的其他文件系统，除非被明确告知跳过）然后将它找到的所有文件名存入数据库

xargs

用来对一系列文件执行给定的命令

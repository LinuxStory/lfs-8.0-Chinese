6.47. Gettext-0.19.8.1

Gettext 软件包包含来用于国际化和本地化的工具。 这些工具用于让程序具有 NLS (本地语言支持）， 使它们能使用用户的本地语言来输出消息。

预计构建时间: 2.9 SBU
需要磁盘空间: 199 MB

6.47.1. Gettext 安装过程

首先，关闭两个在一些机器上会导致无限循环的 test-lock 调用: 

sed -i '/^TESTS =/d' gettext-runtime/tests/Makefile.in &&
sed -i 's/test-lock..EXEEXT.//' gettext-tools/gnulib-tests/Makefile.in

准备编译 Gettext: 

./configure --prefix=/usr    \
            --disable-static \
            --docdir=/usr/share/doc/gettext-0.19.8.1

编译软件包: 

make

　要测试编译结果（这将花费很长时间，大约 3 SBUs), 输入: 

make check

安装软件包: 

make install
chmod -v 0755 /usr/lib/preloadable_libintl.so

6.47.2. Gettext 的内容

安装的程序: autopoint, envsubst, gettext, gettext.sh, gettextize, msgattrib, msgcat, msgcmp, msgcomm, msgconv, msgen, msgexec, msgfilter, msgfmt, msggrep, msginit, msgmerge, msgunfmt, msguniq, ngettext, recode-sr-latin, and xgettext
安装的库: libasprintf.so, libgettextlib.so, libgettextpo.so, libgettextsrc.so, and preloadable_libintl.so
安装的目录: /usr/lib/gettext, /usr/share/doc/gettext-0.19.8.1, and /usr/share/gettext

简要介绍

autopoint

复制标准的 Gettext 基础文件到源代码包

envsubst

替换掉 shell 格式字符串中的环境变量　

gettext

　通过在消息类别中查找翻译来将自然语言消息翻译成用户的语言

gettext.sh

　主要作为 gettext 的一个 shell 函数库

gettextize

　将所有的标准 Gettext 文件复制到制定的软件包的顶级类别以开始 　将它国际化

msgattrib

　在一个翻译类别中根据消息的属性过滤消息并操作这些属性

msgcat

　将给出的 .po 文件连接并合并起来

msgcmp

比较两个 .po 文件 来检查它们都包含同样的 msgid 集合的字符串

msgcomm

　在指定的 .po 文件中找到 　共有的消息

msgconv

　将一个翻译类别转换到不同的字符编码

msgen

　创建一个英文翻译类别

msgexec

　将一个命令应用到一个翻译类别下的所有翻译

msgfilter

　将一个过滤器应用到一个翻译类别下的所有翻译

msgfmt

　从一个翻译类别中生成一条二进制消息类别

msggrep

　将一个翻译类别中符合指定模式或属于某些指定源文件的所有消息提取出来

msginit

　创建一个新的 .po 文件, 　并使用用户环境中的值初始化它的元信息

msgmerge

　将两条原始翻译合并到一个单独的文件

msgunfmt

　将一个二进制消息类别反编译成原始的翻译文本

msguniq

　去除一个翻译类别中的重复翻译

ngettext

　显示一条语法形式基于数字的消息的本地语言翻译

recode-sr-latin

　将塞尔维亚文本从西里尔字母转化为拉丁字母

xgettext

　在指定源文件中提取可翻译消息行来制作第一个翻译模板

libasprintf

定义 autosprintf 类, 这个类让使用了 <string> 和 <iostream> 的 C++ 程序可以 使用 C 格式化输出函数。

libgettextlib

　一个供许多 Gettext 程序使用的包含了公用函数的私有库；这些都 　不是供一般使用的

libgettextpo

　用于写定制程序来处理 .po 文件; 当随 Gettext 附送的标准程序(例如 msgcomm, msgcmp, msgattrib, 和 msgen) 不够用时，可以使用这个库

libgettextsrc

　一个供许多 Gettext 程序使用的包含了公用函数的私有库；这些都 　不是供一般使用的

preloadable_libintl

　一个被 LD_PRELOAD 使用来帮助 libintl 记录未翻译消息的库

6.33. Bash-4.4

Bash 软件包，包含 Bourne-Again SHell。

预计构建时间： 1.7 SBU
所需磁盘空间： 56 MB

6.33.1. 安装 Bash

合并一些上游的补丁： 

patch -Np1 -i ../bash-4.4-upstream_fixes-1.patch

准备编译 Bash： 

./configure --prefix=/usr                       \
            --docdir=/usr/share/doc/bash-4.4 \
            --without-bash-malloc               \
            --with-installed-readline

新选项的含义：

--with-installed-readline

这个选项让 Bash 使用已安装的 readline 库，而不是使用它自带的版本。

编译软件包： 

make

如果不需要运行测试套件的话，跳至“安装软件包”部分继续。

如要准备测试，请确保 nobody 用户能够写入源文件树： 

chown -Rv nobody .

现在以 nobody 用户的身份运行测试： 

su nobody -s /bin/bash -c "PATH=$PATH make tests"

安装软件包，并将主要的可执行文件移至 /bin: 

make install
mv -vf /usr/bin/bash /bin

运行新编译的 bash（替换当前正在执行的那个）： 

exec /bin/bash --login +h

注意

这些参数使 bash 运行一个交互式的登录 shell，并暂时禁用文件校验，这样新文件一旦就位就能使用了。

6.33.2. Bash 包含内容

安装的程序： bash，bashbug，和 sh （链接到 bash）
安装目录： /usr/share/doc/bash-4.4

简介

bash

一个被广泛使用的命令解释器；在执行特定命令行之前，它能够进行多种拓展和替换，这从而使之成为一个强大的工具。

bashbug

一个 shell 脚本，帮助用户撰写并发送标准格式的 bash 错误报告。

sh

一个链接到 bash 的符号链接；若通过 sh 调用，bash 在启动时会尽量模仿老版本 sh 的行为，以便符合 POSIX 标准。

6.22. Acl-2.2.52

Acl 软件包包含了管理访问控制列表的实用程序，用于为文件和目录定义更细粒度的自主访问权限

预计编译时间: 少于 0.1 SBU
所需磁盘空间: 4.8 MB

6.22.1. 安装 Acl

修改文档目录使其成为一个版本化的目录: 

sed -i -e 's|/@pkg_name@|&-@pkg_version@|' include/builddefs.in

修正一些中断的测试: 

sed -i "s:| sed.*::g" test/{sbits-restore,cp,misc}.test

此外，修正在使用getfacl -e时，因过长组名导致段错误的缺陷: 

sed -i -e "/TABS-1;/a if (x > (TABS-1)) x = (TABS-1);" \
    libacl/__acl_to_any_text.c

准备编译 Acl 软件包: 

./configure --prefix=/usr    \
            --bindir=/bin    \
            --disable-static \
            --libexecdir=/usr/lib

编译该软件包: 

make

Acl 测试需要运行在Coreutils 连同 Acl 库被构建之后支持访问控制的文件系统之上。如果需要，返回该软件包并且在Coreutils 被构建之后（本章后文介绍）运行make -j1 tests .

安装该软件包: 

make install install-dev install-lib
chmod -v 755 /usr/lib/libacl.so

该共享库需要被移动到 /lib, 并且需要在 /usr/lib 目录下生成.so 文件: 

mv -v /usr/lib/libacl.so.* /lib
ln -sfv ../../lib/$(readlink /usr/lib/libacl.so) /usr/lib/libacl.so

6.22.2. Acl 的内容

安装的程序: chacl, getfacl, and setacl
安装的库: libacl.so
安装的目录: /usr/include/acl and /usr/share/doc/acl-2.2.52

简要描述

chacl

更改一个文件或者目录的访问控制列表

getfacl

获取文件的访问控制列表

setacl

设置文件的访问控制列表

libacl

包含用于操作访问控制列表的库函数

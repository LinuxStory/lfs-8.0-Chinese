6.50. Coreutils-8.26

Coreutils 软件包包含了显示和设置基本系统特性的工具

预计构建时间: 3.1 SBU
需要磁盘空间: 173 MB

6.50.1. Coreutils 安装过程

POSIX 要求 Coreutils 中的程序即使在多字节 locales 中也能正确识别字符边界。 下面这个补丁修正了这个不兼容和其他的国际化相关的缺陷。 

patch -Np1 -i ../coreutils-8.26-i18n-1.patch

注意

在过去，这个补丁中发现了许多缺陷。当你要向 Coreutils 维护者报告新缺陷时， 请先检查他们在没有打补丁时是否会重现。

关闭一个在一些机器上可能会无限循环的测试。 

sed -i '/test.lock/s/^/#/' gnulib-tests/gnulib.mk

现在准备编译 Coreutils: 

FORCE_UNSAFE_CONFIGURE=1 ./configure \
            --prefix=/usr            \
            --enable-no-install-program=kill,uptime

configure 选项的意义:

FORCE_UNSAFE_CONFIGURE=1

这个环境变量运行软件包以 root 用户身份构建。

--enable-no-install-program=kill,uptime

这个选项用于阻止 Coreutils 安装之后其他软件包会安装的二进制文件。

编译软件包: 

FORCE_UNSAFE_CONFIGURE=1 make

如果不执行测试套件就调到下面 “安装软件包”

现在测试套件已经准备好运行了。首先，需要用户作为 root 来执行测试套件: 

make NON_ROOT_USERNAME=nobody check-root

之后的测试我们需要作为 nobody 用户执行。然而部分测试需要用户是不止一个用户组的成员。 因此我们需要添加一个临时组然后让用户 nobody 成为该组成员，这些测试就不会被跳过了: 

echo "dummy:x:1000:nobody" >> /etc/group

修正一些权限使得非 root 用户也可以编译和运行这些测试： 

chown -Rv nobody .

现在执行这些测试。确保在 su 环境的 PATH 环境变量包含了 /tools/bin. 

su nobody -s /bin/bash \
          -c "PATH=$PATH make RUN_EXPENSIVE_TESTS=yes check"

已知 test-getlogin 测试在一个虚拟终端例如现在的 chroot 环境中会失败，不过它在一个 X 终端中执行会通过测试。

删除临时组： 

sed -i '/dummy/d' /etc/group

安装软件包: 

make install

将程序移动到 FHS 指定的路径： 

mv -v /usr/bin/{cat,chgrp,chmod,chown,cp,date,dd,df,echo} /bin
mv -v /usr/bin/{false,ln,ls,mkdir,mknod,mv,pwd,rm} /bin
mv -v /usr/bin/{rmdir,stty,sync,true,uname} /bin
mv -v /usr/bin/chroot /usr/sbin
mv -v /usr/share/man/man1/chroot.1 /usr/share/man/man8/chroot.8
sed -i s/\"1\"/\"8\"/1 /usr/share/man/man8/chroot.8

LFS-Bootscripts 软件包中的部分脚本依赖于 head, sleep, 和 nice 命令. 因为在启动的早期阶段 /usr 可能不存在。 这些二进制文件需要放在根分区： 

mv -v /usr/bin/{head,sleep,nice,test,[} /bin

6.50.2. Coreutils 的内容

安装的程序: [, base32, base64, basename, cat, chcon, chgrp, chmod, chown, chroot, cksum, comm, cp, csplit, cut, date, dd, df, dir, dircolors, dirname, du, echo, env, expand, expr, factor, false, fmt, fold, groups, head, hostid, id, install, join, link, ln, logname, ls, md5sum, mkdir, mkfifo, mknod, mktemp, mv, nice, nl, nohup, nproc, numfmt, od, paste, pathchk, pinky, pr, printenv, printf, ptx, pwd, readlink, realpath, rm, rmdir, runcon, seq, sha1sum, sha224sum, sha256sum, sha384sum, sha512sum, shred, shuf, sleep, sort, split, stat, stdbuf, stty, sum, sync, tac, tail, tee, test, timeout, touch, tr, true, truncate, tsort, tty, uname, unexpand, uniq, unlink, users, vdir, wc, who, whoami, and yes
安装的库: libstdbuf.so
安装的目录: /usr/libexec/coreutils

根据　base64 (RFC 4648) 编码和解码数据

chroot

date

dd

使用指定的块大小和块数复制文件，可选在复制时执行转换

df

报告所有已挂载文件系统（或仅仅是包含指定文件的文件系统）的磁盘可用空间（和已用空间）

dir

列出每个指定目录的内容 (与 ls 命令相同）

dircolors

dirname

将一个文件名的非目录后缀删除

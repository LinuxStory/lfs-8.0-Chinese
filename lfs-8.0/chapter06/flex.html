6.30. Flex-2.6.3

Flex 软件包包含了一个可生成程序的工具，用于识别文本中的模式(pattern)。

预计构建时间: 0.4 SBU
所需磁盘空间: 32 MB

6.30.1. 安装 Flex

构建过程假定 help2man 程序可用，它可从执行命令的 --help 选项输出中生成帮助页面。然而该程序并不存在，所以我们使用一个环境变量来跳过这个过程。现在，准备编译 Flex: 

HELP2MAN=/tools/bin/true \
./configure --prefix=/usr --docdir=/usr/share/doc/flex-2.6.3

编译该软件包: 

make

如需测试 (约 0.5 SBU), 运行: 

make check

已知其中一个测试 cxx_restart会失败.

安装该软件包: 

make install

一些程序并不知道 flex 并且试图运行其前身 lex. 为了支持这些程序, 创建一个名为lex 的符号链接，实际上是运行了 flex 的 lex 仿真模式: 

ln -sv flex /usr/bin/lex

6.30.2. Flex 的内容

安装的程序: flex, flex++ (link to flex), and lex (link to flex)
安装的库: libfl.so and libfl_pic.so
安装的路径: /usr/share/doc/flex-2.6.3

简要描述

flex

一个用来生成一些程序的工具，这些程序用于识别文本中的模式; 它考虑到了功能的多用途性，可指定模式发现的规则，也就无需为此再开发特定的程序。

flex++

一个 flex 拓展功能，用来生成 C++ 代码和类，是flex的符号链接

lex

一个令flex 以 lex 仿真模式运行的脚本。

libfl

flex 类库

5.12. Expect-5.45

Expect 软件包包含了一个与其他交互式程序进行脚本对话的程序。

大概编译时间： 0.1 SBU
所需磁盘空间： 4.3 MB

5.12.1. 安装 Expect

首先，强制 Expect 的 configure 配置脚本使用 /bin/stty 而不是宿主机系统里可能存在的 /usr/local/bin/stty 这样可以保证我们的测试套件工具在工具链的最后一次能够正常构建： 

cp -v configure{,.orig}
sed 's:/usr/local/bin:/bin:' configure.orig > configure

现在准备编译 Expect： 

./configure --prefix=/tools       \
            --with-tcl=/tools/lib \
            --with-tclinclude=/tools/include

配置选项含义：

--with-tcl=/tools/lib

这个选项可以保证 configure 配置脚本会从临时工具目录里找 Tcl 的安装位置， 而不是在宿主机系统中寻找。

--with-tclinclude=/tools/include

显式地告诉 Expect 去哪搜索 Tcl 内部头文件。通过这个选项可以避免 configure 脚本不能自动发现 Tcl 头文件位置的情况。

编译： 

make

现在编译完成了。之前已经说过，不要求为本章中所构建的临时工具运行测试套件。然而你还是想测试 Expect 的话，可以用下面的命令： 

make test

注意在某些主机上，运行 Expect 测试套件可能会出现测试失败的情况，这是我们所不能控制的。不过，在这里测试套件运行失败也并不奇怪，而且也不是关键所在。

安装软件包： 

make SCRIPTS="" install

make 参数的含义：

SCRIPTS=""

这个变量可以避免安装额外的 Expect 脚本。在这里，我们不需要这些脚本。

5.12.2. Expect 软件包内容

安装的程序： expect
安装的库： libexpect-5.45.so

简介

expect

基于脚本和其他交互式程序通信

libexpect-5.45.so

包含一些允许 Expect 用作 Tcl 扩展或直接用于 C 或 C++（不用 Tcl）的函数
